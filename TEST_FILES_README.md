# Unifyt Testing Files - Complete Guide

## 📋 Overview

This directory contains comprehensive testing files for the unifyt package, specifically addressing the abbreviation confusion between `'m'` (meter) and `'min'` (minute).

## 🎯 The Problem & Solution

### ❌ The Problem
```python
Quantity('10 s', 'm')  # ERROR: Cannot convert from s to m
```
- You were using `'m'` thinking it means "minute"
- But `'m'` actually means "meter" (length)
- Cannot convert time (seconds) to length (meters)!

### ✅ The Solution
```python
Quantity('10 s', 'min')  # ✓ Works: 0.1667 min
```
- Use `'min'` for minute
- Use `'m'` for meter
- No code changes needed - just use correct abbreviations!

## 🎓 Best Practices

1. **Use standard abbreviations**
   - Refer to ABBREVIATION_GUIDE.md
   - When in doubt, use full names

2. **Check dimensions**
   - Time converts to time
   - Length converts to length
   - Cannot mix dimensions!

3. **Test your conversions**
   - Run test suite before deploying
   - Add your own test cases to CSV

4. **Be explicit**
   - Use `'min'` for minute, not `'m'`
   - Use `'meter'` or `'m'` for meter

5. **Use compound units correctly**
   - `'km/hr'` for kilometers per hour
   - `'m/s'` for meters per second
   - `'kg/m3'` for kilograms per cubic meter

## 🔧 Troubleshooting

### Error: "Cannot convert from s to m"
**Problem:** Using `'m'` for minute
**Solution:** Use `'min'` for minute

### Error: "incompatible dimensions"
**Problem:** Trying to convert between different dimensions (e.g., time to length)
**Solution:** Check that both units are the same dimension

### Test fails in Excel
**Problem:** Expected value doesn't match actual
**Solution:** 
1. Check if abbreviation is correct
2. Verify expected value is accurate
3. Check for rounding differences
